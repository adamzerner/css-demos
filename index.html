  4. Positioning
    1. static
      1. block
        1. Can set width
        2. Can set height
        3. Takes up its own line even when it'd fit on the previous line
        4. When there's a mix of block and inline children, an anonymous block is generated to contain the inline children
      2. inline
        1. Can't set width
        2. Can't set height
        3. Line boxes are generated
          1. line-height
          2. vertical-align
          3. text-align
        4. When split over multiple lines
          1. Applies to only the first and last generated boxes
            1. Horizontal padding
            2. Horizontal margins
            3. Horizontal borders
          2. Applies to all generated boxes
            1. Vertical padding
            2. Vertical borders
    2. relative
      1. Moves the element
        1. Block
        2. Inline
      2. top/right/bottom/left
        1. Is relative to the top-left coordinate of where the element otherwise would be
        2. Percentages are of the containing block
      3. Negative values work
      4. Other elements think that the relatively positioned element is in it's old location
        1. Other elements will overlap the relatively positioned element
        2. Container doesn't get expanded
    3. absolute
      1. Containing block
        1. Not necessarily the parent
        2. Closest positioned ancestor
        3. <html> when there are no positioned elements
      2. top/right/bottom/left
        1. Relative to the corresponding edge
        2. Percentages are relative to the containing block
    4. fixed
      1. Same as absolute except that the containing block is always the viewport
